## <a name="build-17093"></a>Сборка 17093
Общие сведения о Windows в сборке 17093 см. в [блоге Windows](https://blogs.windows.com/windowsexperience/2018/02/07/announcing-windows-10-insider-preview-build-17093-pc/).

#### <a name="important"></a>Внимание!
При запуске WSL в первый раз после обновления до этой сборки необходимо выполнить некоторые действия по обновлению каталогов файловой системы Linux. Это может занять несколько минут, поэтому WSL может показаться медленно запущенным. Это должно происходить только один раз для каждого дистрибутива, установленного из магазина.
* Улучшена поддержка чувствительности к регистру в Дрвфс.
    * Дрвфс теперь поддерживает учет регистра в каталоге. Это новый флаг, который можно задать для каталогов, чтобы указать, что все операции в этих каталогах должны обрабатываться с учетом регистра, что позволяет даже приложениям Windows правильно открывать файлы, отличающиеся только регистром.
    * В Дрвфс есть новые варианты подключения, контролирующие чувствительность к учету регистра для каждого каталога.
        * вариант = Force: все каталоги обрабатываются с учетом регистра (за исключением корневого диска). Новые каталоги, созданные с помощью WSL, помечаются с учетом регистра. Это поведение прежних версий, за исключением пометки новых каталогов с учетом регистра.
        * Case = Dir: только каталоги с флагом учета регистра для каждого каталога обрабатываются с учетом регистра; другие каталоги не чувствительны к регистру. Новые каталоги, созданные с помощью WSL, помечаются с учетом регистра.
        * Case = Off: только каталоги с флагом учета регистра для каждого каталога обрабатываются с учетом регистра; другие каталоги не чувствительны к регистру. Новые каталоги, созданные с помощью WSL, помечаются как нечувствительные к регистру.
    * Примечание. каталоги, созданные WSL в предыдущих выпусках, не имеют установленного флага, поэтому не будут обрабатываться с учетом регистра, если используется параметр "Case = Dir". Можно установить этот флаг в существующие каталоги в ближайшее время.
    * Пример подключения с этими параметрами (для существующих дисков необходимо сначала отключиться, прежде чем можно будет подключиться с разными параметрами): Sudo Mount-t дрвфс C:/МНТ/к-o Case = dir
    * Пока что параметр Case = Force по умолчанию все еще используется. В будущем это будет изменено на case = dir.
* Теперь вы можете использовать косую черту в путях Windows при подключении Дрвфс, например Sudo Mount-t дрвфс//сервер/шаре/МНТ/шаре
* WSL теперь обрабатывает файл/etc/fstab во время запуска экземпляра [GH 2636].

  * н Значение представляет собой список путей. В WSL это список, разделенный двоеточием. В Win32 это список, разделенный точкой с запятой.
  * ть Значение должно включаться только при вызове WSL из Win32
  * Белая Значение должно включаться только при вызове Win32 из WSL
