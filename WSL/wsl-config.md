# <a name="manage-and-configure-windows-subsystem-for-linux"></a>Управление подсистемой Windows для Linux и ее настройка

> Применяется к обновлениям Windows 10 для дизайнеров и более поздних версий.  Ознакомьтесь с нашим обновленным [руководством по установке](./install_guide.md) , чтобы опробовать новые функции управления и запустить несколько дистрибутивов Linux из Microsoft Store.

## <a name="ways-to-run-wsl"></a>Способы запуска WSL

Существует множество способов запуска Linux с подсистемой Windows для Linux.

1. `[distro]`Например`ubuntu`
1. `wsl.exe` или `bash.exe`
1. `wsl [command]` или `bash -c [command]`

Какой метод следует использовать, зависит от того, что вы делаете.

### <a name="launch-wsl-by-distribution"></a>Запустить WSL по распределению

Запуск распространения с помощью приложения дистрибутив запускает это распространение в собственном окне консоли.

![Запуск WSL из меню "Пуск"](media/start-launch.png)

Это то же самое, что и при нажатии кнопки "запустить" в Microsoft Store.

![Запуск WSL из Microsoft Store](media/store-launch.png)

Можно также запустить распространение из командной строки, выполнив `[distribution].exe`команду.

Недостаток выполнения дистрибутива из командной строки заключается в том, что он автоматически изменит рабочий каталог из текущего каталога на домашний каталог дистрибутива.

**Например**

```console
PS C:\Users\sarah> pwd

Path
----
C:\Users\sarah

PS C:\Users\sarah> ubuntu

scooley@scooley-elmer:~$ pwd
/home/scooley
scooley@scooley-elmer:~$ exit
logout

PS C:\Users\sarah>
```

### <a name="wsl-and-wsl-command"></a>WSL и WSL [команда]

Лучший способ запуска WSL из командной строки — использовать `wsl.exe`.

**Например**

```console
PS C:\Users\sarah> pwd

Path
----
C:\Users\sarah

PS C:\Users\sarah> wsl

scooley@scooley-elmer:/mnt/c/Users/sarah$ pwd
/mnt/c/Users/sarah
```

Текущий рабочий каталог `wsl` не только сохраняется, но и позволяет выполнять одну команду с командами Windows.

**Например**

```console
PS C:\Users\sarah> Get-Date

Sunday, March 11, 2018 7:54:05 PM

PS C:\Users\sarah> wsl
scooley@scooley-elmer:/mnt/c/Users/sarah$ date
Sun Mar 11 19:56:57 DST 2018
scooley@scooley-elmer:/mnt/c/Users/sarah$ exit
logout

PS C:\Users\sarah> wsl date
Sun Mar 11 19:55:47 DST 2018
```

**Например**

```console
PS C:\Users\sarah> Get-VM

Name            State CPUUsage(%) MemoryAssigned(M) Uptime   Status

Ubuntu          Off   0           0                 00:00:00 Opera...
Ubuntu (bionic) Off   0           0                 00:00:00 Opera...
Windows         Off   0           0                 00:00:00 Opera...
```

## <a name="managing-multiple-linux-distributions"></a>Управление несколькими дистрибутивами Linux

### <a name="windows-10-version-1903-and-later"></a>Windows 10 версии 1903 и более поздних версий

Вы можете использовать `wsl.exe` для управления дистрибутивами в подсистеме Windows для Linux (WSL), включая список доступных дистрибутивов, настройку распределения по умолчанию и удаление дистрибутивов.

Каждый дистрибутив Linux независимо управляет собственными конфигурациями. Чтобы просмотреть команды, относящиеся к распространению `[distro.exe] /?`, выполните команду.  Например: `ubuntu /?`.

#### <a name="list-distributions"></a>Вывод списка распределений

`wsl -l`,`wsl --list`  
Список доступных дистрибутивов Linux, доступных для WSL.  Если распространение указано, оно будет установлено и готово к использованию.

`wsl --list --all`   
Список всех дистрибутивов, включая те, которые сейчас не используются.  Возможно, они находятся в процессе установки, удаления или находятся в неработающем состоянии.  

`wsl --list --running`   
Список всех распределений, выполняемых в данный момент.

#### <a name="set-a-default-distribution"></a>Настройка распределения по умолчанию

По умолчанию используется дистрибутив WSL, который выполняется при выполнении `wsl` в командной строке.

`wsl -s <DistributionName>`, `wsl --setdefault <DistributionName>`

Задает для распределения по умолчанию значение `<DistributionName>`.

**Например**  
`wsl -s Ubuntu`в качестве дистрибутива по умолчанию устанавливается Ubuntu.  Теперь, когда я `wsl npm init` запускаю, он будет запущен в Ubuntu.  При запуске `wsl` откроется сеанс Ubuntu.

#### <a name="unregister-and-reinstall-a-distribution"></a>Отмена регистрации и повторная установка распространения

Хотя дистрибутивы Linux можно устанавливать через Microsoft Store, их нельзя удалить в магазине.  WSL config позволяет отменить регистрацию или удаление дистрибутивов.

Отмена регистрации также позволяет переустановить дистрибутивы.

> **Внимание!** После отмены регистрации все данные, параметры и программное обеспечение, связанные с этим распределением, будут безвозвратно утеряны.  При переустановке из хранилища будет установлена чистая копия дистрибутива.

`wsl --unregister <DistributionName>`  
Отменяет регистрацию дистрибутива в WSL, чтобы его можно было переустановить или очистить.

Например: `wsl --unregister Ubuntu` приведет к удалению Ubuntu из дистрибутивов, доступных в WSL.  При запуске `wsl --list` он не будет перечислен.

Чтобы переустановить, найдите дистрибутив в Microsoft Store и нажмите кнопку "запустить".

#### <a name="run-as-a-specific-user"></a>Запуск от имени определенного пользователя

`wsl -u <Username>`, `wsl --user <Username>`

Запустите WSL от имени указанного пользователя. Обратите внимание, что пользователь должен существовать в дистрибутиве WSL.

#### <a name="run-a-specific-distribution"></a>Запуск определенного распределения

`wsl --d <DistributionName>`, `wsl --distribution <DistributionName>`

Выполнение указанного распределения WSL можно использовать для отправки команд в определенное распределение без необходимости изменения значения по умолчанию.

### <a name="versions-earlier-than-windows-10-version-1903"></a>Версии более ранние, чем Windows 10 версии 1903

WSL config (`wslconfig.exe`) — это программа командной строки для управления дистрибутивами Linux, работающими в подсистеме Windows для Linux (WSL).  Он позволяет перечислить доступные дистрибутивы, настроить распределение по умолчанию и удалить дистрибутивы.

Хотя WSL config полезен для параметров, охватывающих или координирующих распределения, каждый дистрибутив Linux независимо управляет собственными конфигурациями.  Чтобы просмотреть команды, относящиеся к распространению `[distro.exe] /?`, выполните команду.  Например: `ubuntu /?`.

Чтобы просмотреть все доступные параметры для вслконфиг, выполните команду:`wslconfig /?`

```console
wslconfig.exe
Performs administrative operations on Windows Subsystem for Linux

Usage:
    /l, /list [/all] - Lists registered distributions.
        /all - Optionally list all distributions, including distributions that
               are currently being installed or uninstalled.
    /s, /setdefault <DistributionName> - Sets the specified distribution as the default.
    /u, /unregister <DistributionName> - Unregisters a distribution.
```

## <a name="set-wsl-launch-settings"></a>Задать параметры запуска WSL

> **Доступно в сборке Windows Insider Build 17093 и более поздних версий**

Автоматическая настройка определенных функций в WSL, которые будут применяться при каждом запуске подсистемы с помощью `wsl.conf`. 

Сейчас сюда входят параметры автоподключения и конфигурация сети.

`wsl.conf`находится в каждом дистрибутиве Linux в `/etc/wsl.conf`. Если файл отсутствует, его можно создать самостоятельно. WSL обнаружит наличие файла и прочитает его содержимое. Если файл отсутствует или имеет неправильный формат (т. е. неправильное форматирование разметки), WSL будет продолжать запускаться как обычная.

Ниже приведен пример `wsl.conf` файла, который можно добавить в дистрибутивов:

```console
# Enable extra metadata options by default
[automount]
enabled = true
root = /windir/
options = "metadata,umask=22,fmask=11"
mountFsTab = false

# Enable DNS – even though these are turned on by default, we’ll specify here just to be explicit.
[network]
generateHosts = true
generateResolvConf = true
```

### <a name="configuration-options"></a>Параметры конфигурации

В соответствии с соглашениями. ini ключи объявляются в разделе. 

WSL поддерживает два раздела: `automount` и `network`.
